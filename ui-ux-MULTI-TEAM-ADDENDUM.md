```javascript
{
  user: {
    id: 15,

    profileComplete: true
  },

  teams: [
    {
      id: 1,
      name: "Team Sigma",
      role: "ADMIN",
      jerseyNumber: 7,
      primaryRole: "BATTER"
    },
    {
      id: 2,
      name: "Weekend Warriors",
      role: "COORDINATOR",
      jerseyNumber: 12,
      primaryRole: "ALL_ROUNDER"
    }
  ],

  currentTeamContext: {
    teamId: 1,  // null means "All Teams" view
    role: "ADMIN"
  },

  events: {
    upcoming: [...],
    past: [...],
    created: [...]
  }
}
```

**Context Switching Logic:**
```javascript
// When user switches team in header
function handleTeamSwitch(teamId) {
  setCurrentTeamContext(teamId);
  // Refresh all team-specific data
  fetchTeamStats(teamId);
  fetchTeamEvents(teamId);
  fetchTeamMembers(teamId);
  // Save preference
  localStorage.setItem('lastSelectedTeam', teamId);
}

// On app load
function initializeTeamContext() {
  const lastTeam = localStorage.getItem('lastSelectedTeam');
  if (lastTeam && userTeams.includes(lastTeam)) {
    setCurrentTeamContext(lastTeam);
  } else {
    setCurrentTeamContext(userTeams[0]); // First team
  }
}
```

### 13. Key UX Patterns

#### Pattern 1: Team Context Awareness
**Every page shows which team context you're in**
- Header displays current team
- Cards show team badges
- Actions filtered by role in that team

#### Pattern 2: Role-Based UI
**Different users see different options**
```javascript
{role === 'ADMIN' && <ManageTeamButton />}
{role === 'COORDINATOR' && <CreateEventButton />}
{role === 'PLAYER' && <RSVPButton />}
```

#### Pattern 3: Aggregate vs Specific
**Users can view data at two levels**
- "All Teams" â†’ Aggregate stats, all events
- Specific team â†’ Team-only data, filtered events

#### Pattern 4: Quick Switching
**Minimize friction for multi-team users**
- Swipe down â†’ Team selector
- Last selected team saved
- Team badge always visible
